## Why claims need classification

AI-generated lessons can sound authoritative even when their factual statements lack sources. Before publishing any generated content, every claim that depends on external evidence must be classified and either linked to a source or flagged for review.

This lesson walks you through that process using a single applied task: take a set of claims from a generated draft, classify each one, map it to evidence, and decide whether to publish, verify, or remove it.

## The three claim states

Every factual claim in generated content falls into one of three states:

| State | Meaning | Action |
|-------|---------|--------|
| **verified** | A cited, reviewed source supports the claim. | Publish with citation. |
| **verify** | The claim is plausible but lacks a source. | Add a citation task to the source plan. |
| **unsupported** | No credible source is likely to exist. | Remove or rewrite the claim. |

## Applying the classification

Here is a fragment from a generated lesson draft. Read each numbered claim and classify it.

<CodeBlock code={`# Generated Draft — "Getting Started with MDX"

1. MDX lets you use JSX components inside Markdown files.
   → Source: MDX documentation (https://mdxjs.com/)

2. MDX is the most popular documentation format in 2026.
   → No source provided.

3. MDX files compile to plain HTML with zero runtime overhead.
   → No source provided. Likely inaccurate (MDX uses a JS runtime).
`} language="markdown" filename="draft-claims.md" />

Walk through each claim:

- **Claim 1** has a cited source from the official MDX documentation. If the source has been reviewed, classify as **verified**.
- **Claim 2** makes a popularity assertion with no evidence. Popularity changes over time, so this is plausible but unverified. Classify as **verify** and add a citation task.
- **Claim 3** makes a technical assertion that contradicts how MDX works. No credible source is likely to support it. Classify as **unsupported** and remove it.

<Callout title="Source discipline">
  Use <Citation id="ref-mdx" href="https://mdxjs.com/" quality="official-docs">official MDX documentation</Citation> as a primary source for MDX syntax claims. Mark any claim that goes beyond what the documentation states.
</Callout>

## Mapping claims to evidence

After classifying, map each claim to its evidence trail:

| Claim | State | Evidence | Decision |
|-------|-------|----------|----------|
| MDX supports JSX in Markdown | verified | ref-mdx (official docs) | Publish with citation |
| MDX is the most popular format | verify | Source plan task SP-12 | Hold until sourced |
| MDX compiles to plain HTML | unsupported | Contradicts MDX runtime model | Remove |

<VerifyClaim status="verify">
  Claims about current tool popularity or ecosystem adoption should be checked against recent sources before publication.
</VerifyClaim>

## Summary

Every generated claim needs a classification before publication. Map verified claims to their citations, send verify claims to the source plan, and remove unsupported claims. This three-state model keeps uncertainty visible so reviewers can focus on substance rather than guessing which statements are sourced.
